
# Prompt Engineering
 
## 1. 基本的提示词技巧
1. 详细的描述
2. 让模型充当某个角色
3. 使用分隔符表明输入的不同部分，如"[],{}"
4. 对任务指定步骤
5. 提供例子

## 2. 提示词范式
- **Zero-shot**
在提示词工程中，Zero-shot 指不给示例，直接让模型完成任务。它依赖模型已有能力和任务描述的清晰度。

- **Few-shot Learning**
在提示词工程中，Few-shot 指在提示词里提供少量输入/输出示例，让模型模仿示例格式和判断标准。One-shot 是只给一个示例的特殊情况。

示例：

```
请判断下面文本的情绪，只输出：正面 / 中性 / 负面。

文本：这个产品太好用了。
答案：正面

文本：物流速度一般，包装还可以。
答案：中性

文本：{input}
答案：
```

## 3. JSON(JavaScript Object Notation)
- 本质：符合特定语法的**纯字符串**
- 是一种轻量级的数据交换格式，易于人阅读和编写，同时也易于机器解析和生成。
- JSON使用**键值对**的形式来表示数据结构，支持多种数据类型，如字符串、数字、布尔值、数组和对象等。
- JSON广泛应用于Web开发中，用于在客户端和服务器之间传输数据, 结构化文本

### 1. json对象 < = > python字典   
关键区别在于：json对象的键必须是字符串，而python字典的键可以是任意不可变类型（如字符串、数字、元组等）。
另外，JSON 字符串必须使用双引号，不能使用 Python 字典常见的单引号。
在Python中，可以使用内置的**json模块**来处理JSON数据。以下是一些常用的操作：
```
# Python对象 → JSON字符串
json.dumps(
    字典或列表对象，
    ensure_ascii=False,  # 不要将非ASCII字符转义为\uXXXX格式，可以直接输出中文等非ASCII字符
)

# JSON字符串 → Python对象(字典或列表)
json.loads(JSON字符串)
```

补充：继承于 `BaseModel` 类的子类，有一个方法 `model_dump()`，可以输出类对应的**字典**。

## 4. 让模型稳定输出结构化结果

只在提示词里写“请输出 JSON”并不总是稳定，模型可能会加解释文字或返回非法 JSON。更稳妥的做法：

1. 明确字段名、字段类型和可选值。
2. 要求“只输出 JSON，不要 Markdown 代码块，不要解释”。
3. 在程序侧使用 `json.loads()` 或 LangChain 的 `JsonOutputParser` 校验。

示例：

```
请从文本中提取金融事件，只输出 JSON，不要解释。
格式：
{
  "company": "公司名称，字符串",
  "event_type": "融资/并购/财报/其他",
  "amount": "金额，无法确定则为 null"
}

文本：{text}
```
